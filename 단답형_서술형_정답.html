1. **블랙박스 테스트 설계**
- 동등 분할:  
  - 입력: 10 → 결과: 미성년자  
  - 입력: 25 → 결과: 성인  
- 경계값 분석:  
  - 입력: 18 → 결과: 미성년자  
  - 입력: 19 → 결과: 성인  

2. **소프트웨어 아키텍처 중요성과 특징**
- 중요성: 품질 속성, 이해관계자 요구 반영, 개발 효율성  
- 특징:  
  - 모듈성(Modularity): 변경 용이  
  - 확장성(Scalability): 기능 추가에 유리  
  - 유지보수성(Maintainability): 이해 용이

3. **SRP에 따른 Order 클래스 재설계 예시**
- Order: 주문 정보 저장  
- PaymentProcessor: 결제 처리  
- InventoryChecker: 재고 확인  
- ShippingManager: 배송 정보 관리

4. **소프트웨어 품질 비용**
- 예방 비용: 리뷰, 교육  
- 평가 비용: 테스트  
- 실패 비용: 오류 수정  
- 설명: 초기에 예방/평가 투자 → 결함 증폭 감소 → 총 비용 절감

5. **Function Point Estimation**
A. FP = (기능별 가중치 총합 × 복잡도 계수)  
B.  
- Pros: 요구사항 기반, 언어 독립  
- Cons: 초기 정보 부족 시 부정확, 경험 필요
